# User Personas

## 1. 主要角色 (Primary Persona): Alex - 高效生活追求者

| 特徵項目 | 描述 |
|---|---|
| **角色名稱** | Alex (艾力克斯) |
| **職業背景** | 繁忙的專業人士或熱愛探索生活的現代人 |
| **核心動機** | 減少花在瑣碎行程規劃與重複確認上的時間，確保不錯過任何重要活動或時限。 |
| **技術熟悉度** | 高。習慣使用手機、社交媒體與多種 APP 處理生活事務。 |
| **主要痛點** | 資訊過於分散（Email、LINE、IG 截圖），難以統一管理與追蹤時限（如：搶票開賣、付款截止）。 |
| **對 AI 的期待** | 期待 AI 能夠「主動」提醒並「聰明」地處理資訊，但在關鍵決策（如金流操作）上必須有最終控制權。 |

### 1.1 使用情境範例
- **情境 A (搶票)**: 在工作開會時，需要 AI 幫忙盯著演唱會的抽選結果，並在需要付款前傳送智慧提醒。
- **情境 B (聚餐)**: 看到 IG 上的一家熱門餐廳截圖，隨手傳給 Agent 要求「幫我訂下週五的位子」。
- **情境 C (旅遊)**: 準備去東京旅遊，希望 AI 檢查現有行程是否漏掉了關鍵的交通接駁，並自動產生一份懶人行程包。

## 2. 角色與功能映射 (Persona-Feature Mapping)

| 模組 | Alex 的關注點 |
|---|---|
| 活動搶票 | 確保不漏掉開賣與付款時間，且流程符合網站規範。 |
| 餐廳訂位 | 能夠處理多模態輸入（如語音、截圖），且能自動同步到行事曆。 |
| 旅遊安排 | 具備邏輯檢查（缺口檢查）與備案管理，減輕旅遊前的焦慮。 |
| 智慧提醒 | 提醒內容必須具備「下一步操作」的指引，而不只是單純的通知。 |
