# User Stories - Ticket Anywhere (10 Modules Edition)

## 1. 活動搶票與售票追蹤 (Module 1)
### US-01: 售票生命週期監控與協助
**作為一名** Alex，
**我想要** 系統幫我追蹤特定活動的開賣、抽選與付款時限，並在授權下協助購票，
**以便於** 確保在繁忙時仍能順利購得熱門門票。
- **驗收準則**:
  - [ ] 系統能從文字描述中提取活動名稱、預算、票價偏好與張數。
  - [ ] 支援追蹤開賣、抽選、付款與候補等關鍵時序。
  - [ ] 在關鍵時限前發出具備行動指引（Actionable）的智慧提醒。
  - [ ] **合規性**: 協助購票流程必須符合平台條款，不涉及暴力破解。

## 2. 餐廳訂位管理 (Module 2)
### US-02: 多模態輸入與自動訂位同步
**作為一名** Alex，
**我想要** 透過網址、截圖或語音描述訂位需求，並由 AI 協助查詢與執行訂位，
**以便於** 快速完成聚餐安排並同步至行事曆。
- **驗收準則**:
  - [ ] 支援解析主流訂位平台網址與餐廳資訊截圖 (OCR)。
  - [ ] 語音辨識需能提取日期、人數、時間與特殊要求（如：約會需求）。
  - [ ] 訂位成功後自動產出行事曆事件，並保存訂位代碼與取消政策。

## 3. 旅遊行程安排 (Module 3)
### US-03: 旅遊行程規劃與協作
**作為一名** Alex，
**我想要** 描述旅遊天數與目的地偏好，由 AI 生成每日行程並支援群組協作（投票/協調），
**以便於** 快速產出符合眾人期待的旅行計畫。
- **驗收準則**:
  - [ ] AI 能根據距離、營業時間與天氣優化每日路線。
  - [ ] 支援設定備選方案（如：若該景點沒開，則改去附近的 B 點）。
  - [ ] 提供介面供多人參與者針對候選方案進行投票或確認。

## 4. 個人行事曆與提醒中樞 (Module 4)
### US-04: 智慧提醒與任務中樞
**作為一名** Alex，
**我想要** 一個整合所有安排的行事曆中樞，且提醒內容能具備上下文意識（Context-aware），
**以便於** 我在對的時間知道該做什麼動作。
- **驗收準則**:
  - [ ] 提醒內容應包含任務背景（如：建議 11:50 登入，因為 12:00 開賣）。
  - [ ] 行事曆應整合「AI 執行狀態」與「待確認事項」的視覺化標記。

## 5. 智慧任務轉換 (Module 5)
### US-05: 行程缺口分析與補齊
**作為一名** Alex，
**我想要** 系統主動檢查目前行程中的邏輯缺口（如：有機票但沒交通、有住宿但沒晚餐），
**以便於** 透過一鍵操作補齊遺漏的安排。
- **驗收準則**:
  - [ ] 系統能掃描旅遊計畫，識別交通、門票或餐飲的缺失。
  - [ ] 補齊建議應符合Alex的偏好記憶（如：偏好步行或特定菜系）。

## 6. Agentic 規劃機制 (Module 6)
### US-06: 複雜任務自動拆解
**作為一名** Alex，
**我想要** 系統將一個大目標（如：安排五天四夜東京行）自動拆解為多個可執行的子任務，
**以便於** 我能清晰看到 Agent 的執行路徑。
- **驗收準則**:
  - [ ] 系統應顯示任務計畫（Task Plan），列出所有預計執行的子步驟。
  - [ ] Alex 可以針對特定子步驟進行修改或重新排序。

## 7. 偏好記憶 (Module 7)
### US-07: 個人化習慣學習與代入
**作為一名** Alex，
**我想要** 系統學習並記住我的座位偏好、預算範圍與飲食限制，
**以便於** 每次建立新任務時能自動優化建議。
- **驗收準則**:
  - [ ] 系統自動在對話中提示：「已根據您的演唱會偏好（靠走道）進行優化」。
  - [ ] 提供「偏好管理單頁」，讓 Alex 能手動微調 AI 學習到的記憶。

## 8. 人類確認機制 (Module 8)
### US-08: 高風險操作強制授權
**作為一名** Alex，
**我想要** 系統在執行金流交易、取消訂單或重要變更前，必須顯示明確的確認視窗，
**以便於** 我能掌握所有具備法律或財務效力的動作。
- **驗收準則**:
  - [ ] 所有的訂購/付款/取消動作必須具備「兩段式確認」流程。
  - [ ] 確認視窗必須詳列關鍵資訊（如金額、不可退改條款）。

## 9. 任務狀態追蹤 (Module 9)
### US-09: 任務儀表板與狀態管理
**作為一名** Alex，
**我想要** 一個可視化的看板來追蹤所有任務的進度（例如：監控中、執行中、失敗待處理），
**以便於** 快速掌握全局並處理異常情況。
- **驗收準則**:
  - [ ] 看板應支援標記「等待補充資訊」與「等待使用者核准」。
  - [ ] 系統應能自動整理 Email/訊息中的確認信，並更新任務狀態。

## 10. 自動產生旅行行程包 (Module 10)
### US-10: 離線行程包整合生成
**作為一名** Alex，
**我想要** 在旅遊出發前，由系統生成一份整合憑證、地圖與每日流程的完整行程包，
**以便於** 在國外沒有網路或緊急情況下仍能順利進行。
- **驗收準則**:
  - [ ] 支援匯出為離線 PDF 格式。
  - [ ] 內容應包含所有確認號、地址、Google Maps 連結與預算估算。
